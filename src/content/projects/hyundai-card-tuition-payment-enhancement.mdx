## 프로젝트 개요

현대카드 대학등록금 납부 시스템의 프로세스를 고도화하여, 카드사 결제 프로세스상 등록금과 필수/기타 납입금을 한 번의 승인을 하지 못하는 제약사항을 개선했습니다. 이를 통해 학생들이 필수/기타 납입금을 납부하지 않고 지나치는 상황을 방지하고, 추가 납부가 필요한 경우 명확하게 안내하는 UX를 구현했습니다.

## 프로젝트 정보

- **고객사**: 현대카드, 숭실대학교
- **포지션**: Front/Back-end
- **개발 인원**: 1명

## 기술 스택

- **Frontend**: JavaScript, jQuery
- **Backend**: Java, Spring Framework
- **Database**: Oracle

## 업무 프로세스

### 핵심 문제

카드사 결제 프로세스상 등록금과 필수/기타 납입금을 한 번의 승인을 하지 못함에 따라, 학생들이 필수/기타 납입금을 납부하지 않고 지나치는 상황 발생

### 납부 시나리오

**초기 납부 시나리오**

1. **등록금만 존재**
   - 추가 납입금 없이 납부 완료

2. **등록금 + 필수 납입금 존재**
   - 등록금 납부 후 필수 납입금 추가 납부 안내

3. **등록금 + 필수 납입금 + 기타 납입금 존재**
   - 등록금 → 필수 납입금 → 기타 납입금 순차 납부 안내

4. **등록금 + 기타 납입금 존재**
   - 등록금 납부 후 기타 납입금 추가 납부 안내

**해결 방안**: 추가 납부를 인지할 수 있도록 레이어 UI 구성

### 등록금만 납부 후 브라우저 종료 시 재진입 시나리오

1. **필수 납입금 존재**
   - 납부 페이지 진입 시 필수 납입금 미납 안내

2. **필수 납입금 + 기타 납입금 존재**
   - 납부 페이지 진입 시 필수/기타 납입금 미납 안내

3. **기타 납입금 존재**
   - 납부 페이지 진입 시 기타 납입금 미납 안내

**해결 방안**: 납부 페이지 진입 시 추가 납부 안내

## 담당 업무

### 1. **등록금 납부 프로세스 화면 전환에 따른 상태값 관리**

- **납부 상태 추적 시스템 구축**
  - 등록금, 필수 납입금, 기타 납입금 각각의 납부 상태 관리

- **납부 순서 및 플로우 제어**
  - 등록금 → 필수 납입금 → 기타 납입금 순차 진행 로직
  - 각 단계별 결제 완료 여부 검증
  - 미납 항목이 있을 경우 다음 단계 진행 차단

- **브라우저 종료 후 재진입 처리**
  - DB에 납부 이력 저장
  - 재진입 시 미납 항목 자동 조회 및 안내
  - 중복 결제 방지 로직 구현

### 2. **추가 납부를 인지할 수 있는 레이어 UI 구현**

- **모달 레이어 팝업 설계**
  - jQuery를 활용한 모달 레이어 UI 구현
  - 추가 납부 필요 항목 강조 표시
  - 납부 금액 및 항목명 명확하게 표시

- **사용자 인터랙션 처리**
  - "계속 납부하기" / "나중에 납부하기" 선택 옵션 제공
  - 선택에 따른 다음 단계 분기 처리
  - 닫기 버튼 클릭 시 재안내 로직

- **납부 안내 메시지 최적화**
  - 납부 시나리오별 맞춤 안내 문구 작성
  - 필수 항목과 선택 항목 구분 표시
  - 총 납부 금액 및 남은 금액 표시

## 개발 구현 내용

### Frontend 개발

1. **jQuery 기반 납부 프로세스 UI**
   - 단계별 납부 진행 상황 표시 인터페이스
   - 레이어 팝업을 통한 추가 납부 안내
   - 납부 항목별 체크리스트 UI

2. **상태 관리 로직**
   - AJAX를 통한 실시간 납부 상태 조회

3. **모달 레이어 UI 컴포넌트**
   - jQuery 기반 모달 팝업 구현
   - 애니메이션 효과 및 오버레이 처리

### Backend 개발

4. **Spring Framework 기반 서비스 레이어**
   - Spring MVC 패턴
   - 납부 상태 관리 서비스 구현

5. **납부 프로세스 비즈니스 로직**
   - 등록금/필수/기타 납입금 구분 로직
   - 납부 순서 검증 및 제어
   - 결제 완료 후 상태 업데이트 처리
   - 미납 항목 조회 및 안내 로직

6. **결제 연동 및 검증**
   - 현대카드 결제 모듈 연동
   - 결제 승인/취소 처리
   - 결제 이력 관리 및 로깅

### Database 개발

7. **Oracle DB**
   - 등록금 납부 상태 테이블 설계
   - 필수/기타 납입금 항목 관리 테이블
   - 결제 이력 및 상태 추적 테이블
   - 미납 항목 조회 쿼리 최적화

### 프로세스 개선

8. **납부 플로우 최적화**
   - 시나리오별 납부 프로세스 분기 처리
   - 재진입 시 미납 항목 자동 안내 로직
   - 중복 결제 방지 및 에러 핸들링

9. **UX 개선**
   - 명확한 납부 안내 메시지
   - 단계별 진행 상황 시각화
   - 사용자 편의성을 고려한 UI/UX 설계
