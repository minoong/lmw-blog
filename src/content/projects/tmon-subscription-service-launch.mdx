## 프로젝트 개요

티몬의 새로운 비즈니스 모델인 구독서비스를 신규 개발하여 런칭했습니다. 정기적으로 상품을 배송받는 구독 모델을 통해 고객에게 편리함을 제공하고, 안정적인 매출 구조를 구축하는 것이 목표였습니다.

## 기술 스택

- **Frontend**: React, TypeScript
- **State Management**: Redux Toolkit
- **Build Tool**: Webpack
- **Architecture**: Presentational & Container Pattern

## 프로젝트 정보

- **개발 기간**: 2021년 12월 ~ 2022년 2월
- **협업**: 기획자, 마크업 개발자, 백엔드 개발자

## 담당 업무

### 1. 구독 내역 상세

사용자가 구독 중인 서비스의 상세 정보를 확인할 수 있는 페이지를 개발했습니다.

**주요 기능**:
- 구독 상품 정보 표시
- 다음 배송 예정일 표시
- 구독 시작일 및 결제 이력
- 배송지 정보 조회
- 구독 상태 (정상, 일시정지, 해지 등)

### 2. 결제 수단 변경

구독 중인 서비스의 결제 수단을 변경할 수 있는 기능을 구현했습니다.

**주요 기능**:
- 현재 등록된 결제 수단 조회
- 신용카드 추가/변경
- 결제 수단 유효성 검증

### 3. 구독 해지/철회

사용자가 구독을 해지하거나 해지를 철회할 수 있는 기능을 개발했습니다.

**주요 기능**:
- 구독 해지 신청
- 해지 사유 수집
- 해지 예정일 안내
- 해지 철회 (해지 예정일 이전)
- 해지 완료 처리

## 구현 내용

### 1. 프로젝트 보일러플레이트 구축

프로젝트 개발 환경 구축을 위해 Webpack과 TypeScript 설정을 진행했습니다.

**설정 항목**:
- Webpack 설정 (개발/프로덕션)
- TypeScript 설정
- ESLint, Prettier 설정
- 폴더 구조 설계

### 2. Presentational & Container 패턴 적용

UI와 비즈니스 로직을 분리하여 관심사를 명확히 했습니다.

**관심사 분리의 장점**:
- **Container**: API 호출, Redux 연동, 라우팅, 에러 처리
- **Presentational**: props 기반 순수 UI 렌더링
- 재사용성 증가
- 코드 가독성 개선

## 협업 프로세스

### 1. 기획자와의 협업

**업무 협의 내용**:
- 구독 서비스 플로우 정의
- 사용자 시나리오 검토
- 예외 케이스 처리 방안
- 구독 해지 프로세스 설계

**정기 미팅**:
- 기능 명세서 검토 및 피드백
- 화면 플로우 수정 사항 논의

### 2. 마크업 개발자와의 협업

**업무 협의 내용**:
- 컴포넌트 구조 협의

**협업 방식**:
- 퍼블리싱 산출물을 React 컴포넌트로 변환
- HTML/CSS를 JSX로 마이그레이션
- 동적 스타일 처리 (조건부 렌더링)
- 크로스 브라우징 테스트

### 3. 백엔드 개발자와의 협업

**업무 협의 내용**:
- API 스펙 정의 (요청/응답 포맷)
- 에러 코드 정의

**API 문서화**:
- Swagger를 통한 API 문서 공유
- 실시간 API 테스트
- Mock API 서버 활용 (개발 초기)
