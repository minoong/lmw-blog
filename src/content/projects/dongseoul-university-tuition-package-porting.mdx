## 프로젝트 개요

실시간 대학등록금은 학사에서 학생들의 납부 상태를 확인하고, 원장 추가/변경 시 은행에 실시간 EAI 전송 서비스를 할 수 있도록 하는 패키지 시스템입니다. 기존 패키지를 동서울대학교 환경에 맞게 포팅 및 커스터마이징 작업을 수행했습니다.

## 프로젝트 정보

- **개발 기간**: 2016.05 ~ 2016.07
- **고객사**: 국민은행, 동서울대학교
- **포지션**: 패키지 포팅
- **개발 인원**: 1명

## 기술 스택

- **Backend**: Java, Spring Framework
- **Database**: Oracle
- **Server**: Windows
- **Integration**: EAI (Enterprise Application Integration)

## 담당 업무

### 1. **학사/원장 테이블 레이아웃 설계/구현**

- 기존 대학 학사 시스템과 연동을 위한 테이블 스키마 분석
- 학생 정보, 등록금 납부 내역, 원장 정보 테이블 설계
- Oracle 데이터베이스 마이그레이션 및 데이터 매핑 작업
- 테이블 인덱스 최적화 및 쿼리 성능 개선

### 2. **서버 구성에 따른 방화벽 협의**

- 학교 내부 네트워크와 은행 시스템 간 통신 구조 설계
- 방화벽 정책 수립 및 포트 오픈 요청
- 보안 담당 부서와의 협의 및 네트워크 구성 문서 작성
- VPN 및 전용선 연결 검토

### 3. **EAI 전송 테스트**

- 은행 시스템과의 EAI 인터페이스 연동 테스트
- 등록금 원장 데이터 실시간 전송 검증
- 전문 포맷 매핑 및 전송 성공/실패 케이스 테스트
- 에러 핸들링 및 재전송 로직 구현
- 트랜잭션 처리 및 데이터 정합성 검증

### 4. **운영/유지보수**

- 시스템 안정화 및 모니터링
- 사용자 요청사항 대응 및 버그 수정
- 정기 점검 및 성능 튜닝
- 장애 대응 및 로그 분석

## 개발 구현 내용

### Backend 개발

1. **Spring Framework 기반 서비스 레이어**
   - Spring MVC 패턴
   - 학사 시스템 연동 서비스 구현

2. **등록금 납부 관리 시스템**
   - 학생별 납부 상태 조회 기능
   - 납부 내역 관리 및 통계 기능
   - 미납자 리스트 추출 및 관리

3. **원장 관리 및 EAI 연동**
   - 원장 추가/수정/삭제 기능 구현
   - 원장 변경 시 은행 시스템으로 실시간 EAI 전송
   - 전문 생성 및 파싱 로직 구현
   - 전송 이력 관리 및 에러 로그 기록

### Database 개발

4. **Oracle DB**
   - 학생 정보, 등록금 납부 내역 테이블 설계
   - 원장 정보 및 EAI 전송 이력 테이블 설계
   - 프로시저를 통한 배치 처리 및 데이터 정합성 체크

### 시스템 통합 및 운영

5. **EAI 인터페이스 구현**
   - 은행 시스템과의 전문 통신 프로토콜 구현
   - 동기/비동기 방식의 데이터 전송 처리
   - 타임아웃 및 재전송 로직 구현
   - 암호화 및 보안 처리

6. **패키지 포팅 작업**
   - 기존 패키지의 동서울대학교 환경 적용
   - 학사 시스템 데이터 구조에 맞춘 커스터마이징
   - 설정 파일 및 환경 변수 설정
   - 배포 및 운영 환경 구축

7. **운영 환경 구축 및 모니터링**
   - Windows 서버 환경 설정
   - 애플리케이션 배포 및 성능 모니터링
   - 로그 관리 및 장애 대응
