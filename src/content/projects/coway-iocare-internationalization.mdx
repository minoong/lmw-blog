# IoCare+ 글로벌 서비스 국제화 (i18n)

IoCare+ 앱을 6개국(한국, 미국, 중국, 프랑스, 스페인, 말레이시아)에 서비스하기 위한 국제화 유틸리티/커스텀훅을 설계하고 구현했습니다. 각 국가별로 다른 시간 표기 방식, 날짜 형식, 타임존, 단위계를 자동으로 처리하여 사용자에게 높은 만족도를 제공했습니다.

## 기술 스택

- **Next.js 15 App Router** - 서버/클라이언트 컴포넌트 기반 다국어 처리
- **Intl.DateTimeFormat** - 브라우저 네이티브 날짜/시간 포맷팅
- **moment-timezone** - 타임존 변환 및 DST 처리
- **i18next, react-i18next** - 번역 리소스 관리 및 번역 유틸리티/커스텀훅

---

## 1. 로케일별 시간 표기 차이점

각 국가마다 시간을 표기하는 방식이 다릅니다. 특히 **24시간 표기법 vs 12시간 표기법**, **AM/PM 위치**가 차이점입니다.

### 국가별 시간 표기 규칙

| 국가 | 로케일 | 기본 시각 체계 | 오전/오후 표기 | 예시 |
|------|--------|--------------|--------------|------------------|
| 🇰🇷 한국 | `ko-KR` | 12시간제 | **오전 / 오후 (prefix)** | **오전 8:30** |
| 🇺🇸 미국 | `en-US` | 12시간제 | **AM / PM (suffix)** | **8:30 AM** |
| 🇨🇳 중국 | `zh-CN` | 12시간제 | **上午 / 下午 (prefix)** | **上午 8:30** |
| 🇫🇷 프랑스 | `fr-FR` | **24시간제 기본**, 12시간제도 가능 | **a. m. / p. m.** (suffix, 비공식) | **8:30 a. m.** |
| 🇪🇸 스페인 | `es-ES` | **24시간제 기본**, 12시간제도 가능 | **a. m. / p. m.** (suffix) | **8:30 a. m.** |
| 🇲🇾 말레이시아 | `ms-MY` | **12시간제 기본** | **PG / PTG (suffix)** | **8:30 PG** |

### 용어 설명

- **AM (Ante Meridiem)**: 라틴어로 "정오 전", 자정(00:00)부터 낮 12시(11:59) 이전
- **PM (Post Meridiem)**: 라틴어로 "정오 후", 낮 12시부터 자정(23:59) 이전
- **오전/오후**: 한국어 morning/afternoon, 한자로 午前(오전)/午後(오후)
- **上午/下午**: 중국어 morning/afternoon (shangwu/xiawu)
- **PG (Pagi)**: 말레이시아어로 "아침" (AM에 해당)
- **PTG (Petang)**: 말레이시아어로 "오후" (PM에 해당)
- **Prefix**: 시간 앞에 붙이는 표기 (예: 오전 8:30, 上午 8:30)
- **Suffix**: 시간 뒤에 붙이는 표기 (예: 8:30 AM, 8:30 a. m.)
- **24시간 표기법**: 00:00부터 23:59, 군사식 시간 표기(military time)로도 불림
- **12시간 표기법**: 12:00 AM부터 11:59 AM, 12:00 PM부터 11:59 PM

## 3. 구현 범위

### 날짜/시간 포맷팅
- **Intl.DateTimeFormat** 기반 로케일별 시간 포맷팅
- 서비스 정책에 맞춘 12시간제/24시간제 설정
- **moment-timezone** 병행 사용으로 타임존 변환 처리

### 다국어 번역
- **i18next + react-i18next** 기반 번역 유틸리티/커스텀훅
- 6개국 번역 리소스 관리
- 타입 안전한 번역 키 관리

### UI 컴포넌트
- 로케일 인식 시간 선택기 (TimePicker)
- AM/PM 위치 자동 조정 (prefix/suffix)
- 실시간 시간 미리보기

### 타임존 처리

IoCare+ / NECOA 서비스는 **2가지 타임존 개념**을 구분하여 처리합니다.

#### 1. 기기 타임존
- 사용자 디바이스의 타임존
- 사용자가 실제로 위치한 지역의 시간
- 개인 설정, 알림 등에 사용

#### 2. 장소 타임존 (채팅방 개념)
- **장소를 생성한 어드민의 타임존**을 기준으로 설정
- 장소 기준의 시간을 표기할 때 사용
- 예: 디바이스 예약 시간, 장소 내 공유 일정 등
- 여러 국가에 있는 사용자가 같은 장소를 공유할 때 일관된 시간 표기 제공

**구현**: [moment-timezone](https://momentjs.com/timezone/)을 활용하여 서버 UTC 시간을 각 타임존에 맞게 변환

#### 지원 타임존

| 국가 | 타임존 | UTC 오프셋 | DST 적용 |
|------|--------|-----------|---------|
| 🇰🇷 한국 | Asia/Seoul | UTC+9 | 없음 |
| 🇺🇸 미국 | America/New_York | UTC-5 (EST) / UTC-4 (EDT) | 3월~11월 |
| 🇨🇳 중국 | Asia/Shanghai | UTC+8 | 없음 |
| 🇲🇾 말레이시아 | Asia/Kuala_Lumpur | UTC+8 | 없음 |
| 🇫🇷 프랑스 | Europe/Paris | UTC+1 (CET) / UTC+2 (CEST) | 3월~10월 |
| 🇪🇸 스페인 | Europe/Madrid | UTC+1 (CET) / UTC+2 (CEST) | 3월~10월 |

### 단위계 국제화

사용자가 **앱 설정**에서 원하는 단위계를 선택할 수 있도록 구현했습니다.

#### 지원 단위계

| 항목 | 국제단위계 (SI) | 야드파운드법 (Imperial) |
|------|----------------|----------------------|
| 온도 | 23.0°C | 73.4°F |
| 무게 | 5.0 kg | 11.0 lb |
| 부피 | 10.0 L | 2.64 gal |

#### 설정 방식
- 국가에 따라 자동으로 결정되지 않음
- 앱 내 **설정 옵션**에서 사용자가 직접 선택
- 선택한 단위계는 앱 전체에 일관되게 적용

| 앱 설정 화면 |
|-------------|
| ![앱 설정 - 단위계 선택](/projects/IMG_8724.PNG) |
| 사용자가 설정에서 **SI Units** 또는 **Imperial** 선택 |

**사용 예시**: IoT 정수기 필터 용량
- SI 단위계 선택 시: **10.0 L**
- Imperial 단위계 선택 시: **2.64 gal**
